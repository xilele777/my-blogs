如何高效更新本博客

仅供本人参考使用。
March 11, 2026 · 小锅巴

撰写 (Write)#

正常在 content/posts/ 目录下写 Markdown 文章，或者修改已有内容。

预览 (Preview)#

在本地文件夹下的终端运行以下命令，启动本地服务器进行预览：

.\dev.ps1
  • 本地服务器带有实时刷新功能，可通过随时查看修改效果。
  • 确认没问题后，Ctrl + C 停止服务。

发布 (Deploy)#

这是最关键的一步 。已有自动化脚本 deploy.ps1 ，它会自动完成“构建生产环境代码”、“提交到 Git”和“推送到 GitHub Pages”的所有操作。

在终端运行：

.\deploy.ps1
  • 它会自动运行 hugo –minify 生成正确的链接。
  • 它会自动把代码推送到 main 分支。
  • 它会自动把 public 文件夹的内容推送到 gh-pages 分支（GitHub Pages 真正展示的内容）。
  1. 环境分离：dev.ps1 用于在本地调试， deploy.ps1 确保发布出去的代码链接是正确的，指向 GitHub 而不是 localhost。
  2. 防止遗漏 ： deploy.ps1 把 git add , git commit , git push 和复杂的 git subtree push 命令都打包在一起了，避免漏掉某一步（比如忘记推送到 gh-pages 分支导致网站没更新）。
